Networking in GCP


Overview


In Google Cloud Platform (GCP), networking is done differently compared to traditional IP networks. This document aims to explain the basics of networking in GCP.


Subnets in GCP


Subnets are used to segment a network into smaller groups, each with its own IP address range. However, unlike traditional IP networks where subnets fit into an address hierarchy, GCP subnets do not have parent IP address ranges assigned to them.


Communication Between VMs


When considering two virtual machines (VMs) in the same region but different zones, they can still communicate with each other using the same subnet IP address. This is because the first address in a subnet is reserved for the router address and the last address is reserved for the broadcast address.


Google Edge Routers


However, when it comes to communication between VMs across different zones, traffic does not traverse the internet as one might think. Instead, it goes through the Google Edge routers, which have different billing, latency, and security criteria that need to be taken into consideration.


Designing IP Plans in GCP


Typically, IP plans are designed with sub-networks in a top-down hierarchy. However, in GCP, networks do not have IP ranges, so you simply set up a network and create multiple subnets for different departments or teams.


Benefits of Sub-Networks


Sub-networks can provide some rudimentary levels of security and manage resources. However, there are more flexible ways to achieve this using Cloud IAM, labels, tags, and managed instance groups.

Conclusion


Networking in GCP is done differently compared to traditional IP networks. Subnets are used to segment a network into smaller groups, but they do not fit into an address hierarchy like traditional IP networks. Google Edge routers play a crucial role in communication between VMs across different zones. Network Segmentation in GCP


In traditional data centers, IP plans are designed using a top-down hierarchy, with master networks segmented into smaller blocks called subnets. This approach minimizes broadcast traffic and provides rudimentary security between departments.


However, in GCP, this approach is not necessary. Instead, you create multiple subnets for each department, without the need for an IP address range. This allows for more flexibility in network design and management.
