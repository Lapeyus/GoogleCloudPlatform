Logging and Monitoring in GCP


When using containers and microservices with GCP, it's common to use Istio integrated with Stackdriver for monitoring and logging, or Stackdriver alone. However, when considering logs, there are two distinct types: admin data logs and private data logs.


Admin data logs record every action done through the admin user account and are immutable, while private data logs log changes within applications and resources, requiring higher permission levels to access. It's essential to configure these logs carefully, excluding services that don't need monitoring, such as Cloud SQL.


Logging remains within projects created in GCP, which can be problematic for large organizations with many projects. Fortunately, Stackdriver has a log aggregation export feature that allows collating logs from multiple projects into a single log, and filtering and transforming them before sending to other services like BigQuery or Pub/Sub.


Compliance


To ensure compliance, it's crucial to configure object versioning on archived logs in Storage buckets, preventing accidental deletion. Additionally, using the gcloud resource-manager liens create command can prevent project deletion by a malicious third party or accident.
