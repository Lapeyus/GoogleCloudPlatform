Kubernetes


Introduction


Potentially when you use containers a host can scale up or down and start and stop the containers on-demand as demand for your application changes, or if a host fails. But you will need tools to orchestrate and control the provisioning process and that is where Kubernetes comes in.


The Role of Kubernetes


Kubernetes acts as the orchestrator managing many containers deployed on many hosts and scaling them as microservices.
Kubernetes also plays an important role in automatic provisioning and deploying rollouts and rollbacks.


Why We Need Kubernetes


To understand how and why we need Kubernetes as an orchestrator we have to comprehend that deploying and managing containers manually is not really feasible. This is due to fact that there is a lot more too just building and running containers there are the other tasks associated with service discovery, managing updates, monitoring, and provisioning, which are the essential components of an automated, reliable, scalable, distributed system.


Benefits of Kubernetes


To assist us in automating the management and provisioning of our container base we can use Kubernetes, which is an open source orchestrator that abstracts containers at a higher level so you can better manage and scale your applications. Kubernetes can be thought of as a set of APIs that you can use to deploy and manage containers.


Architecture


The Kubernetes system is organised as a set of master components that run as a control plane, and a set of nodes that run containers. In Kubernetes, a node represents a computing instance like a virtual machine. Specifically in Google Cloud, Kubernetes nodes are virtual machines running in the Compute Engine.


Nodes, Pods and Clusters


At a very high level of operations, you describe a set of applications and how they  should interact with each other, and Kubernetes figures how to  make that happen.
Kubernetes uses the concept of a cluster which is basically a set of containers. Hence once you have built your container, the first thing you will need to do is deploy it into a cluster.


Kubernetes Engine (GKE)


GKE is hosted Kubernetes by Google. Within GKE clusters can be customized, they can support different machine types, numbers of nodes, and network settings. Kubernetes is also able to launch containers in existing VMs or to provision new VMs and place the containers within them.


Pod


This is the smallest unit in Kubernetes that you can create or deploy. Logically a pod will represent a running process on your cluster. It can be either a component of your application, or the entire application.


Conclusion


Kubernetes provides an automated way to manage and scale containerized applications, making it easier to deploy and manage complex systems.

Understanding Kubernetes: An Orchestrator for Containerized Applications


Kubernetes acts as an orchestrator managing multiple containers deployed on various hosts and scaling them as microservices. This open-source system abstracts containers at a higher level, enabling better management and scaling of applications.


The Need for Automation in Container Management


Deploying and managing containers manually is not feasible due to the numerous tasks associated with service discovery, updates, monitoring, and provisioning. These components are essential for an automated, reliable, scalable, and distributed system.


Kubernetes as a Solution


Kubernetes can be thought of as a set of APIs that deploy and manage containers. The system consists of master components running as a control plane and nodes running containers. In Google Cloud, these nodes represent virtual machines running in the Compute Engine.


Key Concepts: Nodes, Pods, and Clusters


    

  • Cluster: A cluster is essentially a set of containers.
    • 

  • Pods: Logical groups of similar or tightly coupled containers related to an application. Pods are provisioned within Compute Engine VMs.
    • 

  • Kubernetes Engine (GKE): Hosted Kubernetes by Google, allowing for customization and support for different machine types, nodes, and network settings.

Benefits of Using Kubernetes


    

  • Automates the management and provisioning of containerized applications
    • 

  • Abstracts containers at a higher level for better management and scaling
    • 

  • Provides tools for monitoring and managing containers
    • 

  • Supports automatic provisioning and deploying rollouts and rollbacks
