API Management and Development Environments


Introduction


Managing APIs and development environments is a crucial task in software development. Google Cloud Platform provides two API management tools, Cloud Endpoints and Apigee Edge, to help developers manage their APIs.


Deprecation and Functional Upgrades


Developers need to document changes to APIs, including deprecation or functional upgrades, so that clients and other developers know what's happening. This includes documenting new features in version 2 of an API that are not available in version 1.


Cloud Endpoints


Overview


Cloud Endpoints is a platform for developing and managing API proxies. It implements capabilities such as exposing APIs to trusted developers, monitoring usage, and logging. Cloud Endpoints supports applications running on GCP's compute platforms in various languages and client technologies.


Use Cases


    

  • Exposing APIs to trusted developers while ensuring security and monitoring.
    • 

  • Implementing features like rate limiting, quotas, and analytics.
    • 



Apigee Edge


Overview


Apigee Edge is a platform for developing and managing API proxies with a focus on business problems. It provides capabilities like rate limiting, setting quotas, and providing analytics. However, it requires backend services to be in GCP.


Use Cases


    

  • Decomposing legacy applications into microservices or micro-apps.
    • 

  • Moving monolith applications to the cloud in a less risky way by peeling off services one by one.
    • 



Development Environments


Overview


Developers need to choose between Google App Engine (GAE) and Google Kubernetes Engine (GKE) for building development environments in the cloud. Both provide scalability, redundancy, rollouts, and rollbacks, but differ in management.


Key Differences


    

  • GKE provides fine-grained control over clusters.
    • 

  • GAE manages containers with minimal configuration and management.
    • 



Conclusion


Managing APIs and development environments is essential for software development. Cloud Endpoints and Apigee Edge provide tools for managing APIs, while GAE and GKE offer platforms for building development environments in the cloud. Developers need to choose the right tool based on their specific needs and requirements.

The first approach is related to issues where you're developing a software service and using one of GCP's backends.

The criteria you would like to meet are that you'd like to make it easy to expose this API but also ensure it's only consumed by other developers whom you trust. Thus, you'd like the API to know which end user is making the service call, in addition to being monitored and usage logged.

Cloud Endpoints
In this scenario, you would use Cloud Endpoints as it implements these capabilities and more using a proxy that is easy to deploy in front of your backend services. It also provides an API console to wrap up those capabilities in an easy-to-manage interface. Cloud Endpoints supports applications running in GCP's compute platforms in your choice of languages and client technologies.

Apigee Edge is another platform for developing and managing API proxies, but it has a different approach as it is primarily focused on business problems such as rate limiting, setting quotas, and providing analytics. It is advantageous and less risky to move a monolith application into the cloud by using Apigee Edge to peel off its services one by one, standing up microservices to implement each in turn until the legacy application can be finally retired.

The client likes the idea of building a development environment in the cloud but would like to know which platform to use App Engine or Kubernetes? Solution

The issue here is that both Google Kubernetes Engine (GKE) and Google App Engine (GAE) can be the right choice, depending on the degree of control needed. Both provide benefits such as scalability, redundancy, rollouts, and rollbacks, but differ in their management.

GKE gives you fine-grained control over everything about your cluster, while GAE manages containers with minimal configuration or management, restricting control.
