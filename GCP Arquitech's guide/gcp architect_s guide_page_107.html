Compute Engine's VMs: A Deep Dive


Overview of Virtual Machine Instances


In Google Cloud Platform (GCP), the traditional virtual machine instance is the most commonly used compute and processing option. This is due to Compute Engine's flexibility in creating a VM instance, allowing users to run any OS or language they choose.


Pure IaaS Model


Compute Engine operates on a pure Infrastructure as a Service (IaaS) model, where users receive a VM and an operating system, with full control over management aspects such as auto-scaling and firewall rules.


Primary Workload


The primary workload of Compute Engine is general workloads, especially enterprise applications designed for server infrastructure. This makes it portable and easy to run in the cloud.


Predefined vs. Custom Machine Types


When creating a VM instance, users can choose from predefined or custom machine types. The latter allows users to select memory, CPU, disk type, and networking configuration, as well as run a combination of Linux and Windows machines.


Features for Configuration


In addition to security, networking, and load balancing, Compute Engine offers features such as:


    

  • Availability policies
    • 

  • Pricing options
    • 

  • Usage discounts (e.g., start-up scripts, machine rightsizing, firewall rules)
    • 



Customizing the VM


When discussing VMs, it's essential to consider CPU requirements. Compute Engine provides a range of predetermined machine configurations suitable for most use cases, including micro, medium, high CPU, low to high memory, and standard and shared-core machine types.


However, if these machine types are not suitable, users can customize their own machine by selecting specific parameters such as CPU, memory, disk type, and networking configuration.
