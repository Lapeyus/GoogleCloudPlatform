For example, if software development is Project A, and test engineering is Project B, you can assign different people to different roles in the projects for management separation or what is called separation of duties. To achieve this, you should consider segmenting the system up into multiple projects to enable more granular access control.

Bastion Host

Another common network design makes use of a bastion host to provide internal resources and VMs with isolation from the internet. Bastion hosts provide an external facing point of entry into a network containing private network instances.

In this scenario, only the bastion host is configured with a public external IP address on its interface, while all other internal VM instances use the private internal address space addresses. The bastion host is therefore acting like a proxy or gatekeeper for internet access for both ingress and egress traffic.

This bastion host model can also be deployed to provide a single point for fortification or for network audit purposes, which is controllable as its SSH services can be either started or stopped to permit or block any inbound SSH connections coming from the Internet. For example, should you have an application or service running on an internal VM instance that is provided to an internal corporate audience but requires remote administrative access, then there's no need to provide this instance with an external IP address.

Network Isolation Models for Resource Security


Overview of Network Isolation


Network isolation is a crucial aspect of resource security, preventing the compromise of one part from spreading to other parts. This can be achieved through various models, including:


    

  • Resource Isolation: Resources are isolated to prevent communication between them over private IP addresses.
    • 

  • Communication Via Public IPs: VMs communicate via external IP addresses across the Internet.
    • 

  • Bastion Host: A bastion host provides internal resources and VMs with isolation from the internet.
    • 



Resource Isolation Models


Resource isolation models involve isolating resources to prevent communication between them. This can be achieved through:


    

  • VPC Network Peering: Allowing resources to communicate over a private address space.
    • 

  • Separate Projects: Isolating VMs into separate projects, but within the same zone.
    • 



Communication Via Public IPs


Communication via public IPs involves assigning different people to different roles in projects for management separation or separation of duties. This can be achieved by:


    

  • Segmenting the system up into multiple projects.
    • 

  • Using separate projects implies that VMs must communicate via external IP addresses across the Internet.
    • 



Bastion Host Model


The bastion host model provides internal resources and VMs with isolation from the internet. A bastion host is configured with a public external IP address on its interface, while other internal VM instances use private internal address space addresses.


    

  • Single Point for Fortification: The bastion host can be deployed as a single point for fortification or network audit purposes.
    • 

  • Controllable SSH Services: The bastion host's SSH services can be started or stopped to permit or block any inbound SSH connections coming from the Internet.
