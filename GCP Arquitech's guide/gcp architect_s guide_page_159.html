Overview


To establish a VPN does not require any hardware devices in the Google Cloud as there is no actual VPN device on the Google side as any compatibility issues running your VPN between your existing it is purely software defined. Benefits for Small Organisations

After all it is secure and as it connects over the network it can be configured and setup anywhere in the world where there is internet access.


Limitations


A VPN is a good starting point for a secure interconnect as it is very convenient or at least to start with. However, there are some significant issues that may arise when you start to grow your GCP network and then interconnect it with your on-premises network.


Network Routing Issues


The thorny issue here is that of network routing. For traffic to flow between subnets on both sides of the VPN they need to know about one another's existence. This is easy to start off with as you will only need to exchange subnet information for a few subnets on configuring a few static routes to let the routing process know that either side of the VPN tunnel.


However, as the two networks gradually merge then static routes to subnets become unwieldy and do not scale well. This is because with static routing, updating the tunnel requires the addition of adding all the static routes for each on-premises subnet to GCP. It also requires adding routes for every existing subnet in GCP for the on-premises network. That is bad enough but any new subnet will also need to be added before it can interconnect and the VPN tunnel restarted to include the new subnet. This is not really sustainable.


Cloud VPN Gateways


Google Cloud Platform offers two types of Cloud VPN gateways, HA VPN and Classic VPN.


Classic VPN Gateway


The legacy type is called a classic VPN gateway and it has a single interface, a single external IP address, and it supports tunnels using static routing.


High Availability VPN (HA VPN)


The second type of VPN that you can create is called a High-Availability VPN (HA VPN) and it lets you securely connect your on-premises network to your GCP Virtual Private Cloud network through an IPsec VPN connection in a single region.

Establishing a Secure VPN Connection with Google Cloud Platform


To establish a VPN connection with Google Cloud Platform (GCP), no hardware devices are required, as the process is purely software-defined. This means that compatibility issues and IKE protocol usage are minimized.


Ideal for Small Organizations or Development Environments


A VPN is an ideal starting point for secure interconnects, especially for small organizations or those just starting out with a development VPC to test their waters. It provides security and flexibility in configuring the setup anywhere in the world where internet access is available.


Growing Concerns and Network Routing Issues


However, as GCP networks grow and more organizational functions are moved onto the platform, significant issues arise. One of the main concerns is network routing, which becomes unwieldy and unscaleable with static routing. Updating the tunnel requires adding all static routes for each on-premises subnet to GCP, as well as routes for every existing subnet in GCP.


Google Cloud Platform VPN Gateways


To address these issues, Google Cloud Platform offers two types of Cloud VPN gateways: Classic VPN and High Availability VPN (HA VPN). The Classic VPN gateway supports tunnels using static routing, while the HA VPN gateway provides a secure connection through an IPsec VPN connection in a single region.
